# AST Chunker Integration into ZigZag

## Overview

Integrate the tree-sitter-based AST chunker (currently a standalone C project in `llm/`) into ZigZag's build and LLM report pipeline. The chunker extracts top-level semantic units (functions, classes) as line ranges and uses them to produce more organized, context-efficient LLM output. The existing head+tail condensation (`condenseContent`) is preserved for unsupported languages.

## 2. Top-Level Makefile

Add a `compile_commands` target so C code in `ast/` remains editable with full IDE tooling:

```makefile
compile_commands:
    cmake -S ast -B ast/build
    ln -sf ast/build/compile_commands.json ast/compile_commands.json
```

This is the only CMake invocation that remains; it is a dev convenience, not part of the build.

## 3. `build.zig` Changes

Add AST C sources and include paths to the `mod` module (the zigzag library). `mod.link_libc = true` is already set.

```zig
mod.addCSourceFiles(.{
    .root = b.path("ast"),
    .files = &.{
        "vendor/tree-sitter/src/lib.c",
        "grammars/tree-sitter-python/src/parser.c",
        "grammars/tree-sitter-python/src/scanner.c",
        "src/chunker.c",
    },
    .flags = &.{"-std=c99"},
});
mod.addIncludePath(b.path("ast/vendor/tree-sitter/include"));
mod.addIncludePath(b.path("ast/src"));
mod.addIncludePath(b.path("ast/grammars/tree-sitter-python/src"));
```

Adding to `mod` (not `exe` directly) ensures both the executable and the test binary see the C sources, since `exe_tests` uses `exe.root_module` which imports `mod`.

## 4. Zig Wrapper — `src/cli/commands/report/writers/llm/ast_chunker.zig`

New file. Wraps the C API into a clean Zig interface. Source bytes come from `entry.content` (already in memory), so no C-side file I/O is needed.

**Public API:**

```zig
pub const Chunk = struct {
    start_line: u32, // 0-based (matches tree-sitter output directly)
    end_line: u32,   // 0-based, inclusive
};

/// Returns null if the extension is unsupported — caller falls back to condenseContent.
/// Caller owns the returned slice; free with allocator.free().
pub fn chunkSource(
    source: []const u8,
    ext: []const u8,
    allocator: std.mem.Allocator,
) !?[]Chunk
```

**Internals:**
- `@cImport(@cInclude("chunker.h"))` + `extern fn tree_sitter_python() *const c.TSLanguage`
- Private `languageAndTypes(ext)` maps `.py` → `(tree_sitter_python(), &python_types)`, returns null for all other extensions — single extension point for future grammars
- Line numbers stored 0-based internally; callers add 1 for display
- `c.free_chunk_result` called via `defer` before copying into the Zig-owned slice

## 5. LLM Writer Changes — `llm.zig`

For each file in `real_entries`, attempt AST chunking before falling back to `condenseContent`:

**Supported file (e.g. `.py`):**
- Split `entry.content` into lines
- For each chunk, emit a separate fenced block:
  ```
  ### src/animals.py [1–25]
  ```python
  class Animal:
      ...
  ```
  ```
- File index entry: `- src/animals.py (3 AST chunks)`

**Unsupported file:**
- Existing `condenseContent` path unchanged
- File index entry unchanged: `- src/file.go (condensed — 80 of 200 lines shown)` / `(N lines, full)`

**ChunkWriter (chunked output) path:** No special handling needed — AST chunk blocks flow through `cw.writeFile` / `cw.rotateChunk` the same as today.

## 7. Out of Scope

- AST chunking for languages other than Python (pattern established; grammars added later)
