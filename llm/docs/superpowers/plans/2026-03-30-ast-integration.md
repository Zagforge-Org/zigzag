# AST Chunker Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Integrate the tree-sitter C library into ZigZag's build system and use it to emit AST-chunked output in the LLM report for Python files.

**Architecture:** `llm/` is renamed to `ast/` and its CMake build is replaced by `build.zig` compiling the C sources directly into the zigzag module. A new `ast_chunker.zig` wraps the C API; `llm.zig` uses it to emit per-function/class fenced blocks for `.py` files, falling back to the existing `condenseContent` path for unsupported languages.

**Tech Stack:** Zig 0.15.2, tree-sitter C runtime, tree-sitter-python grammar, `@cImport`/C interop

- [ ] **Step 1: Replace the stub with the full implementation**

Overwrite `src/cli/commands/report/writers/llm/ast_chunker.zig`:

```zig
const std = @import("std");
const c = @cImport(@cInclude("chunker.h"));

extern fn tree_sitter_python() *const c.TSLanguage;

const python_types = [_][*c]const u8{
    "function_definition",
    "class_definition",
    "decorated_definition",
};

pub const Chunk = struct {
    start_line: u32, // 0-based (tree-sitter row)
    end_line: u32,   // 0-based, inclusive
};

const LanguageConfig = struct {
    language: *const c.TSLanguage,
    node_types: []const [*c]const u8,
};

fn languageConfig(ext: []const u8) ?LanguageConfig {
    // entry.extension always has a leading dot — strip it
    const e = if (ext.len > 0 and ext[0] == '.') ext[1..] else ext;
    if (std.mem.eql(u8, e, "py")) {
        return .{
            .language = tree_sitter_python(),
            .node_types = &python_types,
        };
    }
    return null;
}

/// Returns null if the extension is unsupported or no top-level chunks are found.
/// Caller owns the returned slice; free with allocator.free().
pub fn chunkSource(
    source: []const u8,
    ext: []const u8,
    allocator: std.mem.Allocator,
) !?[]Chunk {
    const lc = languageConfig(ext) orelse return null;

    const config = c.ChunkConfig{
        .node_types = @ptrCast(lc.node_types.ptr),
        .node_type_count = @intCast(lc.node_types.len),
    };

    const result = c.extract_chunks(
        lc.language,
        &config,
        source.ptr,
        @intCast(source.len),
    );
    defer c.free_chunk_result(result);

    if (result.count == 0) return null;

    const chunks = try allocator.alloc(Chunk, result.count);
    for (result.chunks[0..result.count], chunks) |raw, *out| {
        out.* = .{ .start_line = raw.start_line, .end_line = raw.end_line };
    }
    return chunks;
}
```

- [ ] **Step 4: Add the `getLineRange` private helper at the bottom of `llm.zig`**

After the closing brace of `writeLlmReport`, add:

```zig
/// Extracts lines [start_line..=end_line] (0-based) from content as a slice.
/// The returned slice points into content — no allocation.
fn getLineRange(content: []const u8, start_line: u32, end_line: u32) []const u8 {
    var line: u32 = 0;
    var i: usize = 0;

    // Advance to start_line
    while (i < content.len and line < start_line) : (i += 1) {
        if (content[i] == '\n') line += 1;
    }
    const start_byte = i;

    // Advance past end_line
    while (i < content.len and line <= end_line) : (i += 1) {
        if (content[i] == '\n') line += 1;
    }
    return content[start_byte..i];
}
```
