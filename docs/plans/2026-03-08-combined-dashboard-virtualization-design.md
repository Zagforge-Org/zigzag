# Combined Dashboard Virtualization — Design

**Date:** 2026-03-08
**Status:** Approved

## Problem

The combined dashboard (`combined-dashboard.html` / `combined.ts`) renders all file rows eagerly at page load. `renderPathSection()` calls `p.files.map(renderFileRow).join("")` for every section, creating the full DOM immediately. On large repositories (e.g. Next.js: 24,666 source files), this freezes the browser.

The single-path dashboard (`table.ts`) already has a working virtual scroll and is unaffected.

## Root Cause

`combined.ts:104` — `const fileRows = p.files.map((f) => renderFileRow(f)).join(""); `
`combined.ts:165` — `container.innerHTML = R.paths.map((p, i) => renderPathSection(p, i)).join("")`

All rows for all sections are built synchronously on load, even for collapsed sections.

## Solution

Replace naive row generation with a per-section `VirtualTable` class, lazily instantiated on first expand. Only ~25 rows are in the DOM at any time per section. Same algorithm as `table.ts`.

## Component 1 — `src/templates/src/virtual-table.ts` (new)

Self-contained `VirtualTable` class encapsulating the virtual scroll state:

- `constructor(onRowClick: (f: CombinedFile) => void)` — builds the `.table-viewport` DOM, attaches scroll listener
- `setFiles(files: CombinedFile[]): void` — swaps the dataset, resets `scrollTop = 0`, re-renders the visible window; bottom spacer height = `(files.length - end) * ROW_H` so large search results (e.g. 15k matches) maintain correct scroll height
- `getElement(): HTMLElement` — returns the `.table-viewport` div, ready to append to the DOM

Internal constants (matching `table.ts`):
- `ROW_H = 40` — fixed row height in px
- `OVERSCAN = 10` — rows rendered beyond the visible edge
- Two spacer `<tr>` rows (top/bottom) for scroll height simulation
- RAF-throttled scroll handler (`rafPending` guard)

Hover prefetch: `mouseover` on `.file-link` spans calls `fetchContent(path, noop)` to warm the cache before the user clicks.

No sorting — keeps the combined table simple; can be added later.

## Component 2 — Modified `combined.ts`

**`renderPathSection()`**: Remove `fileRows` string and `<table class="file-table">` block. Replace with `<div class="vtable-mount"></div>`.

**New module-level map**: `const sectionTables = new Map<string, VirtualTable>()`

**`attachSectionToggle()`**: On every toggle, after `classList.toggle("expanded")`, check if the section is now expanded AND has no entry in `sectionTables`. If so:
1. Create `new VirtualTable(openCombinedViewer)`
2. Call `vtable.setFiles(currentFilteredFiles(pathData))` — reads the active global search query so a pre-filtered view is applied on first open
3. Append `vtable.getElement()` into `.vtable-mount`
4. Store in `sectionTables.set(root_path, vtable)`

Subsequent toggles collapse/expand the section without re-creating the VirtualTable.

**`filterAllSections()`**: For each section, if `sectionTables.has(root_path)`, call `vtable.setFiles(visible)`. If not yet created (never expanded), skip — the VirtualTable will pick up the current query from the global search input on first expand.

**`attachSectionSearch()`**: Same — call `vtable.setFiles(filtered)` if mounted.

**`attachRowListeners()`**: Removed from `combined.ts` (VirtualTable handles delegated click via `addEventListener` on the viewport).

## Component 3 — CSS

`dashboard.css` already provides `.table-viewport { overflow-y: auto; border: ...; }` (shared by both HTML files). Add one targeted rule:

```css
.path-section .table-viewport {
    max-height: 600px;
}
```

This caps section file tables at 600px, triggering the virtual scroll listener.

## Component 4 — Bundling

After TS/CSS changes: `python3 src/templates/bundle.py` regenerates both `dashboard.html` and `combined-dashboard.html`. No Zig source changes needed.

## Performance Impact

| Scenario | Before | After |
|---|---|---|
| Page load (Next.js, 24,666 files) | ~24k+ `<tr>` created | 0 file rows in DOM |
| Expand first section (24,666 files) | Freeze | ~25 rows, instant |
| Global search (15k results) | Rebuilds all tbody innerHTML | `setFiles()` swaps array, spacers update to `15k * 40px` height |
| Collapse/re-expand section | Re-renders all rows | VirtualTable DOM persists, `renderVisible()` called once |
