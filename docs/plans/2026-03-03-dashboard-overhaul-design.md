# Dashboard Overhaul Design

**Date:** 2026-03-03
**Status:** Approved

## Goals

1. **Maintainability** — Break the 812-line monolithic `dashboard.html` into separate source files (CSS, JS, Prism worker) with a Zig build step that bundles them into the final self-contained HTML.
2. **Performance** — Move Prism syntax highlighting off the main thread into a Web Worker. Add virtual scrolling to the file table.
3. **Language coverage** — Expand Prism from 5 to 15+ language grammars.
4. **Dark mode** — Auto dark mode via `prefers-color-scheme`.

## Constraints

- The final output must remain a **single self-contained HTML file** (no CDN, no external assets), because it is embedded via `@embedFile` in `src/cli/commands/report.zig`.
- The `__ZIGZAG_DATA__` and `__ZIGZAG_CONTENT__` injection markers in `report.zig` must be preserved exactly.
- All HTML element IDs and structural classes that `report.zig` tests rely on must be preserved.

## Source Structure (after)

```
src/templates/
  dashboard.html           ← bundled output; committed; regenerated by build step
  src/
    dashboard.css          ← all styles + dark mode overrides
    prism-theme.css        ← Prism Tomorrow Night theme
    dashboard.js           ← app logic (viewer, virtual table, charts, watch mode)
    highlight-worker.js    ← Prism core + all grammars; runs as a Blob Worker
  bundle.py                ← ~50-line Python bundler; replaces <!-- @inject: file --> markers
```

The bundler reads `src/templates/src/template.html` (the shell HTML with `<!-- @inject: ... -->` markers) and outputs `src/templates/dashboard.html`.

## Zig Build Integration

In `build.zig`, a `RunStep` is added before the main compile step:

```zig
const bundle = b.addSystemCommand(&.{ "python3", "src/templates/bundle.py" });
exe.step.dependOn(&bundle.step);
```

This ensures `dashboard.html` is always regenerated before the Zig binary is compiled. `@embedFile` in `report.zig` continues pointing to `src/templates/dashboard.html`.

## Syntax Highlighting Architecture

### Off-thread via Blob Worker

`highlight-worker.js` contains the complete Prism core plus all required language grammars. It is inlined into `dashboard.js` as a template literal string and instantiated as a Blob Worker on first viewer open (lazy creation).

**Protocol:**
- Main → Worker: `{ id: number, code: string, language: string }`
- Worker → Main: `{ id: number, html: string }`

**Cancellation:** A monotonically increasing `highlightToken` counter discards stale results when the user switches files before the worker responds.

**Instant display:** On file open, `vcode.textContent = raw` shows plain text immediately. When the worker responds, `vcode.innerHTML = highlightedHtml` replaces it. No size limit is needed.

### Languages

Current (5): javascript, css, markup, json, clike

Added (15+): typescript, zig, rust, go, python, ruby, java, scss, bash, toml, yaml, markdown, c, cpp, sql

## Virtual Scrolling (File Table)

- Fixed row height: 40px
- On scroll: compute `startIdx = Math.floor(scrollTop / ROW_H)` and `endIdx = startIdx + Math.ceil(viewportH / ROW_H) + OVERSCAN`
- Render only `[startIdx, endIdx]` rows
- Top spacer: `height = startIdx * ROW_H`; bottom spacer: `height = (total - endIdx) * ROW_H`
- Hover prefetch: `mouseover` on rendered file-link spans triggers content prefetch
- Search/sort operate on the full filtered array; virtual rendering is a view concern only

## Dark Mode

```css
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0d1117;
    --surface: #161b22;
    --border: #30363d;
    --text: #e6edf3;
    --muted: #8b949e;
    --accent: #58a6ff;
  }
}
```

No JS required. The viewer panel already uses dark colors and is unaffected.

## What Does NOT Change

- HTML structure (same element IDs, same injection markers)
- `report.zig` and `watch.zig` — no Zig changes except the build step
- Watch mode polling logic
- Content worker (file content loading off-thread)
- Stats cards, language chart, size histogram rendering logic

## Testing Approach

- Existing `writeHtmlReport` tests in `report.zig` continue passing (HTML structure preserved)
- Manual browser testing: open a large file (>10K lines), verify no UI freeze
- Verify dark mode in browser DevTools → Rendering → emulate prefers-color-scheme
- Verify Zig build step runs cleanly: `zig build` should succeed end-to-end
