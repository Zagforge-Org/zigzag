# Virtual Code Viewer Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the current dump-everything-into-one-`<code>` viewer with a virtual-scrolling viewer that renders only visible lines, adds line numbers, caches syntax-highlighted chunks, and fixes the Zig operator regex bug.

**Architecture:** Spacer-based virtual scroll (same pattern as the file table) applied to the source viewer. `openViewer` dispatches to `openSimpleViewer` (< 500 lines / 200 KB, line-number `<table>`) or `openVirtualViewer` (large files, spacer + `.vwindow` of `.vline` divs). A per-file `hlChunkCache` caches 200-line highlighted chunks keyed by chunk index; `requestVisibleChunks` sends only uncached ranges to the Prism worker and patches live DOM spans when results arrive.

**Tech Stack:** Vanilla JS (ES5), Prism worker (existing), Python bundler (existing)

    fetchContent(f.path, function (raw) {
        if (viewerToken !== myToken) return;

function openSimpleViewer(content, lines, langKey, myToken) {
    var body = document.getElementById("viewer-body");
    var numWidth = String(lines.length).length + "ch";

    for (var i = 0; i < lines.length; i++) {

function scheduleVirtualViewerRender() {
    if (virtRafPending || !virtLines) return;
    virtRafPending = true;
    requestAnimationFrame(function () {
        virtRafPending = false;
        if (virtLines) renderVirtualWindow();
    });
}

### Step 3: Add `renderVirtualWindow`

```js
function renderVirtualWindow() {
    if (!virtLines || !virtBodyEl || !virtWindowEl) return;
    var total = virtLines.length;
    var scrollTop = virtBodyEl.scrollTop;
    var viewH = virtBodyEl.clientHeight;

    var start = Math.max(
        0,
        Math.floor(scrollTop / VIEWER_LINE_HEIGHT) - VIEWER_OVERSCAN
    );
    var end = Math.min(
        total,
        Math.ceil((scrollTop + viewH) / VIEWER_LINE_HEIGHT) + VIEWER_OVERSCAN
    );

    virtSpacerTopEl.style.height = start * VIEWER_LINE_HEIGHT + "px";
    virtSpacerBotEl.style.height =
        (total - end) * VIEWER_LINE_HEIGHT + "px";

    var frag = document.createDocumentFragment();
    for (var i = start; i < end; i++) {
        var row = document.createElement("div");
        row.className = "vline";

        var ln = document.createElement("span");
        ln.className = "ln";
        ln.style.width = virtGutterW;
        ln.textContent = i + 1;

        var lc = document.createElement("span");
        lc.className = "lc";
        lc.dataset.line = i;

        // Show cached highlighted line if available, otherwise plain text
        var chunkIdx = Math.floor(i / HL_CHUNK_SIZE);
        var lineInChunk = i - chunkIdx * HL_CHUNK_SIZE;
        if (hlChunkCache[chunkIdx] && hlChunkCache[chunkIdx][lineInChunk] !== undefined) {
            lc.innerHTML = hlChunkCache[chunkIdx][lineInChunk];
        } else {
            lc.textContent = virtLines[i];
        }

        row.appendChild(ln);
        row.appendChild(lc);
        frag.appendChild(row);
    }

    virtWindowEl.innerHTML = "";
    virtWindowEl.appendChild(frag);

    // Highlighting requested in Task 5 — placeholder call
    // if (virtLangKey) requestVisibleChunks(start, end, viewerToken);
}
```

### Step 1: Add `requestVisibleChunks`

Add this function after `renderVirtualWindow`:

```js
function requestVisibleChunks(start, end, token) {
    if (!virtLines || !virtLangKey) return;
    var firstChunk = Math.floor(start / HL_CHUNK_SIZE);
    var lastChunk = Math.floor(Math.max(start, end - 1) / HL_CHUNK_SIZE);

    for (var c = firstChunk; c <= lastChunk; c++) {
        if (hlChunkCache[c] || hlChunkPending[c]) continue;
        hlChunkPending[c] = true;
        (function (chunkIdx, myToken) {
            var chunkStart = chunkIdx * HL_CHUNK_SIZE;
            var chunkEnd = Math.min(virtLines.length, chunkStart + HL_CHUNK_SIZE);
            var chunkText = virtLines.slice(chunkStart, chunkEnd).join("\n");

            highlightAsync(chunkText, virtLangKey, function (html) {
                if (viewerToken !== myToken || !virtLines) return;
                delete hlChunkPending[chunkIdx];
                if (!html) return;

                var hlLines = html.split("\n");
                hlChunkCache[chunkIdx] = hlLines;

                // Patch live .lc spans belonging to this chunk
                if (!virtWindowEl) return;
                var cells = virtWindowEl.querySelectorAll(".lc[data-line]");
                for (var j = 0; j < cells.length; j++) {
                    var lineIdx = parseInt(cells[j].dataset.line, 10);
                    if (Math.floor(lineIdx / HL_CHUNK_SIZE) !== chunkIdx) continue;
                    var lineInChunk = lineIdx - chunkStart;
                    cells[j].innerHTML =
                        hlLines[lineInChunk] !== undefined
                            ? hlLines[lineInChunk]
                            : esc(virtLines[lineIdx]);
                }
            });
        })(c, token);
    }
}
```

## Troubleshooting

**Lines appear at wrong vertical position:** `VIEWER_LINE_HEIGHT` in JS (20) doesn't match `height: 20px` in `.vline` CSS. Verify both values match exactly; use browser DevTools to measure the rendered line height.

**Scroll jumps on rapid scroll:** RAF throttle is already in place. If still jumpy, increase `VIEWER_OVERSCAN` from 15 to 25.

**Highlighting wrong colour on multi-line tokens:** Splitting highlighted HTML by `\n` can produce unbalanced spans for multi-line strings/comments. This is cosmetic and affects only the split line; the browser renders it fine (auto-closes spans). No fix needed.

**Virtual viewer doesn't appear for a file you expect:** Check `lines.length` vs `VIRT_LINE_THRESHOLD` (500) and `content.length` vs `VIRT_BYTE_THRESHOLD` (204800). A file with 490 lines and 180 KB will use the simple viewer.

## Post-Implementation Corrections

These are differences between the original plan and what was actually implemented, discovered during testing. The design doc (`2026-03-03-virtual-viewer-design.md`) has been updated to reflect the final state.

### CSS changes from Task 2

The `td.ln` and `td.lc` padding in the plan included `0.5rem` top/bottom. This caused row heights of ~36px. The final CSS uses **no vertical padding** on table cells:

```css
table.ln-table td.ln {
    padding: 0 1rem 0 1rem;   /* was: 0.5rem 1rem 0.5rem 0.75rem */
}
table.ln-table td.lc {
    padding: 0 1rem 0 0;      /* was: 0.5rem 0.75rem 0.5rem 0 */
}
```

`.vline` requires `overflow: hidden` (not in original plan):
```css
.vline {
    display: flex;
    height: 20px;
    overflow: hidden;   /* ← required — prevents content taller than 20px from corrupting scrollHeight */
    font-size: 13px;
    line-height: 20px;
    font-family: Consolas, Monaco, "Andale Mono", "Ubuntu Mono", monospace;
}
```

`#viewer-body` requires `overflow-anchor: none` (not in original plan):
```css
#viewer-body {
    overflow: auto;
    flex: 1;
    padding: 0.75rem 0;
    overflow-anchor: none;  /* ← critical: prevents CSS Scroll Anchoring feedback loop */
}
```

The `td > * { display: block }` rule from the original file table styles must be scoped to `.table-viewport`:
```css
/* Was: td > * — applied globally, made Prism <span> tokens block-level */
.table-viewport td > * {
    display: block;
    padding: 0.4rem 0.75rem;
}
```

### JS changes from Tasks 3–4

**CRLF normalization** (add to `openViewer` fetchContent callback, before splitting):
```js
var content = (raw || "(binary or empty)").replace(/\r\n/g, "\n").replace(/\r/g, "\n");
```

**Gutter width** must include padding to avoid box-sizing content-area collapse:
```js
// openSimpleViewer — was: String(lines.length).length + "ch"
var numWidth = "calc(" + String(lines.length).length + "ch + 2rem)";

// openVirtualViewer — was: String(lines.length).length + "ch"
var digits = String(lines.length).length;
virtGutterW = "calc(" + digits + "ch + 2rem)";
```

**Pre-computed minWidth** (add to `openVirtualViewer` after setting `virtGutterW`):
```js
var maxLen = 0;
for (var mi = 0; mi < lines.length; mi++) {
    if (lines[mi].length > maxLen) maxLen = lines[mi].length;
}
win.style.minWidth = "calc(" + (digits + maxLen) + "ch + 3rem)";
```

**Pin vwindow height before clearing** (add to top of `renderVirtualWindow` before the `frag` loop):
```js
virtWindowEl.style.height = (end - start) * VIEWER_LINE_HEIGHT + "px";
```

This prevents `scrollHeight` collapsing to zero when `innerHTML = ""` is called mid-render.
